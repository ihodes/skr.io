Skr.io is an API for the storage, display, querying & retrieval of textual data.

We expose a simple interface to your text online. With it you can store, display (in multiple formats), query (with XPath or JSONPath), render into HTML (if it is Markdown), display nicely with syntax highlighting and all (if any format), add and remove metadata, or simply return it as it was posted; in plain text.

Our API docs can be found here. Hosted, of course, using Skr.io. If you like, you can change the extension on the url to .render or .txt and see what happens. In fact, this very page is hosted using Skr.io (serving HTML, of course).

How does this work?

In about a minute or two, you'll see just what you can do with Skr.io.

Most of these commands require an API token and secret; you can authenticate using basic auth. You can get a key and secret by signing up; it's free and easy. You should do it and follow along with real credentials.

To post a simple bit of plain text, you may just post it to the API.

It will return the ID of the text object. With it, you can rule the world.
curl http://<token>:<secret>@www.skr.io/api/v.1/text -dtext="Here's our text."

That's boring, though, so let's just POST some JSON instead.

curl -H"content-type: application/json" http://<token>:<secret>@www.skr.io/api/v.1/text \
      -d"{\"apple\": \"pie\", \"turkey\": {\"and\": \"gravy\", \"numbers\": [1,2,3,4]}}"

Which returns, let's say, the id of xyz123. With this ID, we can do some cool things. First of all, let's make this text public, allowing others to see it who don't have our API key and secret.

curl -X POST http://<token>:<secret>@www.skr.io/api/v.1/text/xyz123/public

Now just anyone can see this text at http://skr.io/text/xyz123. It's easy enough to make it private again, too, if you wish.

Let's query this tiny bit of JSON, with JSONPath. If this were XML, you could query it with XPath instead.

curl http://<token>:<secret>@www.skr.io/api/v.1/text/xyz123?q=$.apple

...which should, naturally, return "pie". This queries can get quite complex, including predicates and other shenanigans. This is a very powerful way to query your text.

Even more information can be found in the API documentation, and more features are being added regularly. The documenation itself was published by posting some text formatted with Markdown with content-type application/markdown, and rendered so nicely by adding a .html extension to the URL (and, obviously, made public so that you can see it. You can check out some other formatting options by replacing the .html with a .txt or .render (or even no extension at all). Other content-types have other extensions which can be applied to them; c.f. the documenation. Again, feel free to email me at isaac@marion.io if you have any question, comments, or requests.
